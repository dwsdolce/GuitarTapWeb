# Override-marker consistency — italic + ` *` everywhere

**Status:** ✅ COMPLETE + USER-VERIFIED all 3 platforms (2026-07-24). Cross-platform (Swift → Python
→ web). Separate from the peak-lifecycle work; one commit per repo.

## Decision
A **manually overridden** peak mode is signalled the SAME way on every surface: the mode label is
**italic** with a trailing **` *`**. The small pencil glyph (`pencil.circle.fill` / `✎`) is dropped —
at annotation size it was unreadable (looked like a dot, not a pencil). The separate **in-range**
indicator (check-circle / triangle for "peak frequency inside its assigned mode's band") is a DIFFERENT
signal and is unchanged.

Rationale: the three apps each marked overrides differently, and even within one app the interactive UI
(glyph) disagreed with the tables/PDF (` *`). One vocabulary, chosen as the one already used by the
text/PDF surfaces.

### Override COLOR bug (2026-07-24) — a standard-mode override showed the AUTO mode's color
Distinct from the label/marker: an override to a predefined mode (e.g. Air→Top) was drawn in the
auto mode's color on several surfaces. The override-color rule is now uniform everywhere: predefined
override → that mode's color; freeform label → user-defined teal; else the auto mode.
- **Swift `peakColor`** (ExportableSpectrumChart): only handled the *freeform* case, fell through to
  auto for a predefined override — fixed. Feeds the callout + summary. **Build verified: BUILD SUCCEEDED.**
- **Python `peak_color`** (exportable_spectrum_chart) — same freeform-only bug, fixed (feeds annotation
  callout + summary). **Python PDF page-1 peak table** (`tap_analysis_results_view`) coloured by the
  auto `peak_modes` map — fixed to resolve the override's color. (The user's report: annotation,
  summary, AND table.)
- **Web** — the PDF table color (`modeLabelColor`) was already override-aware; the chart-image marker
  color (`buildGuitarMarkers`) was auto-only and is now fixed, so all three web surfaces match.
- **Swift summary scope fix:** the Detected Peaks Summary is inside the free function
  `makeExportableSpectrumView`, which cannot call the struct's private `peakModeLabel`/`peakColor`/
  `isManualOverride` — the override logic is inlined there.
